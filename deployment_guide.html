Preparing Your Ubuntu Server

Connect to your Ubuntu server via SSH and install Docker Engine, Docker Compose, and configure the firewall.

1.1 Connect via SSH

Terminal (Local Computer) 
# Connect to your server
ssh azuranyan@192.168.1.10

1.2 Update Ubuntu & Install Docker

Ubuntu Terminal 
# Update system packages
sudo apt update && sudo apt upgrade -y

# Install prerequisites
sudo apt install -y curl ca-certificates gnupg lsb-release git ufw

# Install Official Docker & Docker Compose Plugin
sudo curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Add your user 'azuranyan' to the docker group (avoids needing sudo for docker commands)
sudo usermod -aG docker azuranyan

# Activate docker group permissions immediately
newgrp docker

1.3 Configure UFW Firewall

Ubuntu Terminal 
# Allow essential ports through UFW firewall
sudo ufw allow 22/tcp     # SSH
sudo ufw allow 80/tcp     # HTTP
sudo ufw allow 443/tcp    # HTTPS
sudo ufw allow 8001/tcp   # EcoBank HTTP Port
sudo ufw allow 8080/tcp   # Reverb WebSocket Port

# Enable Firewall
sudo ufw --force enable
sudo ufw status verbose

Build & Launch Docker Containers

Build the multi-stage image (Node Vite compilation + PHP 8.4 FPM + Nginx + Supervisord) and start your application.

4.1 Verify docker-compose.yml

Ensure your docker-compose.yml file looks like this:

docker-compose.yml 
version: '3.8'

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: banksampah-app
    restart: unless-stopped
    ports:
      - "8001:80"      # Web App Port
      - "8080:8080"    # Laravel Reverb WebSockets Port
    environment:
      - APP_NAME=EcoBank
      - APP_ENV=production
      - APP_DEBUG=false
      - APP_URL=https://ecobank.my.id
      - DB_CONNECTION=sqlite
      - DB_DATABASE=/var/www/html/database/database.sqlite
      - SESSION_DRIVER=database
      - CACHE_STORE=database
      - QUEUE_CONNECTION=database
      - REVERB_APP_ID=856535
      - REVERB_APP_KEY=a5d7e6wrtwismkjqn5gn
      - REVERB_APP_SECRET=rb97eg7uumei0zzwhrpi
      - REVERB_HOST=0.0.0.0
      - REVERB_PORT=8080
      - REVERB_SCHEME=http
    volumes:
      - ./database:/var/www/html/database
      - app-storage:/var/www/html/storage/app

  app-storage:

4.2 Build and Run Container

Ubuntu Terminal 
# Create database folder on host to ensure proper SQLite persistence
mkdir -p ./database
touch ./database/database.sqlite
chmod -R 777 ./database

# Build and start container in background
docker compose up -d --build

# Check container status
docker compose ps

# Check real-time logs
docker compose logs -f app

4.3 Run Migrations & Storage Link inside Container

Ubuntu Terminal 
# Execute database migrations inside running container
docker exec -it banksampah-app php artisan migrate --force

# Create storage symlink
docker exec -it banksampah-app php artisan storage:link --force

# Cache configuration & routes for maximum production speed
docker exec -it banksampah-app php artisan config:cache
docker exec -it banksampah-app php artisan route:cache
docker exec -it banksampah-app php artisan view:cache

Cloudflare DNS & SSL Setup (ecobank.my.id)

Connect your purchased domain ecobank.my.id to your Ubuntu server using Cloudflare.

🚀
Why Cloudflare Tunnel? It securely connects your local server (192.168.1.10) directly to Cloudflare edge network with zero open ports on your router and automatic SSL certificates!

Step-by-Step Cloudflare Tunnel Setup:

  1. Log in to Cloudflare Dashboard and add domain ecobank.my.id. Update your domain registrar's Nameservers to Cloudflare.
  2. Go to Zero Trust ➔ Networks ➔ Tunnels ➔ Click Create a Tunnel.
  3. Select Cloudflared and name it ecobank-ubuntu.
  4. Copy the Ubuntu installation command provided by Cloudflare:
Ubuntu Terminal 
# Install cloudflared service on Ubuntu
curl -L --output cloudflared.deb https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
sudo dpkg -i cloudflared.deb

# Run your tunnel connector command (Copy from your Cloudflare dashboard):
sudo cloudflared service install <YOUR_TUNNEL_TOKEN>

Configure Public Hostname Routes in Cloudflare Dashboard:

Subdomain / Hostname Domain Type / Service URL
@ (or www) ecobank.my.id HTTP http://localhost:8001
ws (WebSockets) ecobank.my.id HTTP http://localhost:8080

Step-by-Step Direct DNS Setup:

  1. In your Wi-Fi Router Settings, forward Public Port 80 to 192.168.1.10:8001 and Port 443 to 192.168.1.10:8001. Also forward Port 8080 for WebSockets.
  2. Find your router's Public IP address (e.g. by checking curl ifconfig.me).
  3. In Cloudflare DNS Management, create an A Record:
    • Type: A
    • Name: @ (ecobank.my.id)
    • IPv4 Address: <YOUR_ROUTER_PUBLIC_IP>
    • Proxy Status: Proxied (Orange Cloud)
  4. Under SSL/TLS Settings, set Encryption mode to Full or Flexible.

Automated SQLite Database Backups

Since your database uses SQLite, automated daily file backups ensure zero data loss.

Ubuntu Terminal 
# Create backup script directory
mkdir -p /home/azuranyan/scripts
nano /home/azuranyan/scripts/backup_db.sh

Paste the following script inside backup_db.sh:

/home/azuranyan/scripts/backup_db.sh 
#!/bin/bash
BACKUP_DIR="/home/azuranyan/backups/sqlite"
DATE=$(date +%Y-%m-%d_%H-%M-%S)
mkdir -p "$BACKUP_DIR"

# Copy SQLite database file safely
cp /home/azuranyan/hosted_project/database/database.sqlite "$BACKUP_DIR/ecobank_$DATE.sqlite"

# Delete backups older than 30 days
find "$BACKUP_DIR" -type f -name "*.sqlite" -mtime +30 -delete

echo "Backup created: ecobank_$DATE.sqlite"
Ubuntu Terminal 
# Make script executable
chmod +x /home/azuranyan/scripts/backup_db.sh

# Add daily Cron job at 02:00 AM
(crontab -l 2>/dev/null; echo "0 2 * * * /home/azuranyan/scripts/backup_db.sh >/dev/null 2>&1") | crontab -
